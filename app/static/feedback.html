Use this table to teach ClaireBot what to suppress, what to enforce, and how to phrase expected findings.

How to enter feedback
  • Pick Type: false_alarm to suppress recurring noise, missed_issue to enforce checks that were missed, or correction to improve finding wording.
  • Set Mode using the checkboxes to scope the rule to one or more flows: French Review, English Review, and/or Comparison.
  • Write a clear Issue Pattern and optional Expected Finding; keep entries specific and reusable.
How this is used in workflow
  • Saved rows are stored in the feedback registry and injected into the prompt as structured guidance before LLM review.
  • Rules are mode-scoped and only active rows are applied at runtime.
  • Deterministic hard-coded checks still run first; this registry augments downstream LLM behavior and does not replace those fixed rules.
