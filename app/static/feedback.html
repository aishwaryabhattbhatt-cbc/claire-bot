Feedback

Use these three tables to add focused feedback for false alarms, missed issues, and corrections.

How to enter feedback
  • False Alarm: paste the false finding from CSV output and add a short explanation.
  • Missed Issue: describe what issue was missed and the expected finding text.
  • Correction: paste wrong finding text and the corrected finding text.
  • Use Mode and Status controls in each row to scope and activate/deactivate entries.
How this is used in workflow
  • Saved rows are stored in the feedback registry and injected into the prompt as structured guidance before LLM review.
  • Rules are mode-scoped and only active rows are applied at runtime.
  • Deterministic hard-coded checks still run first; this registry augments downstream LLM behavior and does not replace those fixed rules.
